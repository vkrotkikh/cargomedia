--- layout: task title: Task - Error Page ---

The purpose of this task is to create an error page for www.cargomedia.ch and to give us an idea of how you approach a front-end / design task.. The website is a static website using Jekyll and is publicly hosted on Github (cargomedia.github.io). Feel free to look around and get familiar with Github (if not already).

Problem Description

First of all, we want to give you as much artistic freedom as possible. Your solution may fit the current design - may also work completely independent. Be creative, think 'out of the box', surprise us - the result may be funny, crazy, interactive, artistic or incredibly simple and clean. Feel free to make use of all the possibilities of a modern browser - you don't have to worry about compatibility. Everything working in latest (even canary) Chrome, FF, Safari, IE is fine. Of course you can use code snippets and third party libraries (jquery, three.js, transit, etc.) at your convenience.

Our current solution looks like this - So there is room for improvement.

Requirements

  • Comprehensive HTTP 404 error message
  • Use semantic HTML5/CSS3* syntax

    *If you're using a CSS preprocessor, such as Less or Sass, please include the source files as well.

  • Responsive (Mobile First) Design
  • Spend max. 3h. Complete within a week (two weeks in special circumstances).

Instructions

  • Fork the github project cargomedia.github.io
  • Follow the readme instructions to test the site locally. Need more help? Check Github Help and Jekyll for further information
  • Create a folder with your solution in /task/error-page/solutions/<your-alias>/
  • Make a pull-request when ready

We hope you have fun working on this mini-project and are looking forward to examine your solution.
